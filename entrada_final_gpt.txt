--- HISTORIA DE USUARIO ---
id: HDU-EMP-003
titulo: Prevenir eliminación de empleadas
actor: Administrador del sistema
descripcion: >
  Como administrador del sistema,
  quiero que no se permita  eliminar empleados de género femenino,
  para cumplir con la normativa interna de no eliminar registros históricos sensibles.

criterios_aceptacion:
  - Si se intenta eliminar un empleado con género "Femenino", se debe lanzar una excepción o retornar un error controlado.
  - Si el empleado es de género "Masculino" u otro valor, se permite la eliminación normal.
  - El método de eliminación actual debe ser modificado para incluir esta validación.
  - No debe eliminarse ninguna lógica existente que ya esté implementada en ese método.
  - Se deben mantener todos los métodos actuales sin eliminación ni reemplazo.
  - La modificación debe estar delimitada por comentarios que indiquen claramente el inicio y fin del cambio realizado por la IA.


--- DIAGNÓSTICO Y PLAN DE CLASES ---
--- ANÁLISIS DE CLASES EXISTENTES ---

| Clase Java | Clasificación | Motivo |
|------------|----------------|--------|
| Employee | REUTILIZABLE-AS-IS | Contiene atributo `gender` que es necesario para la validación. |
| EmployeeRepository | REUTILIZABLE-AS-IS | Se utiliza para realizar operaciones CRUD en la base de datos. |
| EmployeeService | MODIFICABLE | La interfaz debe ser modificada para lanzar la nueva excepción en el método `deleteEmployeeById`. |
| EmployeeServiceImpl | MODIFICABLE | Debe ser modificado para incluir la validación antes de eliminar un empleado. |
| EmployeeController | REUTILIZABLE-AS-IS | No necesita modificaciones ya que el cambio se maneja a nivel de servicio. |

--- PLAN DE MODIFICACIÓN ---

1. Clase `EmployeeService`: Modificar la firma del método `deleteEmployeeById` para que lance la nueva excepción `CannotDeleteEmployeeException`.

2. Clase `EmployeeServiceImpl`: Modificar el método `deleteEmployeeById` para que realice la validación del género antes de eliminar un empleado. Si el empleado es de género femenino, debe lanzar la excepción `CannotDeleteEmployeeException`.

3. Crear una nueva clase `CannotDeleteEmployeeException` que extienda de `RuntimeException`. Esta excepción se lanzará cuando se intente eliminar un empleado de género femenino.

4. Crear una prueba unitaria para el método `deleteEmployeeById` de la clase `EmployeeServiceImpl` para asegurar que la nueva lógica funcione correctamente. La prueba debe verificar que se lanza la excepción `CannotDeleteEmployeeException` cuando se intenta eliminar un empleado de género femenino.
