--- ANÁLISIS DE CLASES EXISTENTES ---

| Clase Java | Clasificación | Motivo |
|------------|----------------|--------|
| Employee | REUTILIZABLE-AS-IS | Contiene atributo `gender` necesario para la validación |
| EmployeeRepository | REUTILIZABLE-AS-IS | Se utiliza para la consulta y eliminación de empleados |
| EmployeeService | REUTILIZABLE-AS-IS | Interfaz de servicio que contiene el método a modificar |
| EmployeeServiceImpl | MODIFICABLE | Implementación del servicio donde se debe agregar la validación |
| EmployeeController | REUTILIZABLE-AS-IS | Controlador que utiliza el servicio de empleados, no requiere cambios |
| EmployeeCoreApiApplication | REUTILIZABLE-AS-IS | Clase principal de la aplicación, no requiere cambios |

--- PLAN DE MODIFICACIÓN ---

1. Clase `EmployeeServiceImpl`: Se modificará el método `deleteEmployeeById(Long id)` para agregar la validación requerida. Antes de eliminar el empleado, se consultará el empleado por su id y se verificará si su género es "Femenino". Si es así, se lanzará una excepción `CannotDeleteEmployeeException`. 

2. Clase `CannotDeleteEmployeeException`: Se creará esta nueva clase de excepción para manejar el caso en que se intente eliminar un empleado de género femenino. Esta excepción extenderá de `RuntimeException` y tendrá un constructor que acepte un mensaje de error.

3. Pruebas unitarias: Se deberán crear pruebas unitarias para el método modificado `deleteEmployeeById(Long id)` en `EmployeeServiceImpl`. Estas pruebas deberán cubrir los casos en que el empleado a eliminar es de género femenino, de género masculino u otro, y cuando el empleado no existe.
